# Faraday OS Image

Buildroot recipe that produces the Pi Zero OS image — a minimal, read-only Linux that boots straight into the Faraday signer with no networking, no shell on the framebuffer, and no surface area beyond the camera, display, and buttons.

## What's in the image

- Linux kernel built from Buildroot's Pi Zero defconfig
- BusyBox userland (no full coreutils, no shells exposed at boot)
- The cross-compiled Faraday Rust binary at `/opt/faraday`
- An init script (`/etc/init.d/S99faraday`) that launches it on boot

That's it. No SSH, no Wi-Fi firmware (the chip isn't there anyway), no package manager, no swap, no persistent state. Power off = clean slate.

## Layout

```
opt/
├── build.sh              # Runs inside the Docker container — clones Buildroot,
│                         # stages the Faraday binary into the rootfs overlay,
│                         # invokes Buildroot, copies the image out
├── pi0/                  # Buildroot "external tree" for the Pi Zero target
│   ├── configs/pi0_defconfig   # Top-level Buildroot config
│   ├── board/
│   │   ├── kernel.config       # Linux kernel config
│   │   ├── busybox.config      # BusyBox config
│   │   ├── boot_config.txt     # Pi firmware boot config
│   │   ├── boot_cmdline.txt    # Kernel command line
│   │   ├── post-build.sh       # Hooks before image assembly
│   │   └── post-image.sh       # Hooks after image assembly
│   ├── external.desc           # Buildroot external-tree descriptor
│   └── external.mk
├── buildroot/            # (gitignored) Buildroot source — cloned at build time
├── buildroot_dl/         # (gitignored) Buildroot download cache
├── rootfs-overlay/       # (gitignored) Generated rootfs overlay
└── faraday-bin           # (gitignored) Cross-compiled binary, mounted in by docker-compose
```

## Build

From the repo root:

```bash
# 1. Cross-compile the ARM binary
cargo install cargo-zigbuild
cargo zigbuild --release --target arm-unknown-linux-gnueabihf

# 2. Build the image
docker compose up
```

Or via the `just` shortcuts:

```bash
just arm           # Cross-compile
just image         # Cold build (~30 min — Buildroot fetches and compiles everything)
just image-fast    # Warm rebuild (reuses Buildroot output cache, ~2 min)
```

Output: `images/faraday_os.pi0.img`.

## Flash

```bash
# Find your SD card
diskutil list

# Flash via the just shortcut
just flash DEVICE=/dev/diskN
```

Or manually:

```bash
diskutil unmountDisk /dev/diskN
sudo dd if=images/faraday_os.pi0.img of=/dev/rdiskN bs=4m status=progress
diskutil eject /dev/diskN
```

## Customizing

- **Kernel modules / drivers:** edit `pi0/board/kernel.config` (e.g. via `make linux-menuconfig` after a build), then rebuild
- **Userland packages:** edit `pi0/configs/pi0_defconfig`
- **Boot behavior:** edit `pi0/board/boot_cmdline.txt` (kernel cmdline) or `boot_config.txt` (Pi firmware)
- **Init / startup:** the binary is launched from `/etc/init.d/S99faraday`, which is generated by `build.sh` (search for `INITD`)

After any config change, run `just image-fast` for a quick rebuild. Use `just image` (cold) only when you've changed Buildroot itself or want a clean state.
